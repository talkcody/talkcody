## [0.4.1] - 2026-02-11

### 新增

- **视频输入支持**：新增视频输入功能，可以直接根据视频生成前端页面（当前只有 Kimi API 模型支持，下个版本将支持更多模型）
- **MiniMax M2.5 模型**：新增 MiniMax M2.5 模型支持，并设为默认免费模型
- **GLM-5 模型**：新增 GLM-5 系列模型

### 改动

- 将 Kimi Coding Plan 作为单独的 Provider

### 修复

- **修复 Issue #50**：修复终端面板相关的 bug
- **修复 Issue #51**：修复 workspace root 相关的 bug
- **修复 Issue #52**：修复窗口关闭应用没有完全退出的 bug


## [0.4.0] - 2026-02-09

- **Telegram 远程控制**：支持 Telegram 远程控制桌面的TalkCody，支持文本，语音，图片消息，支持流式返回消息。

- [飞书 远程控制用户文档](docs/content/docs/zh/features/feishu-remote.mdx)
- **语音转写**：新增 Groq Whisper 语音转写支持，可以免费使用。
- **模型支持**：新增 OpenRouter 的 Pony Alpha 免费模型。
- **Explore Agent**：开放 Explore Agent，适合快速了解代码仓库。

- 修复分享页面主题显示问题。

## [0.3.1] - 2026-01-25

### 新增

- **Hooks 系统**：新增事件驱动的 Hooks 机制，支持在关键事件（用户提交、工具调用前后、会话开始/结束、停止等）触发自定义脚本，实现操作拦截、参数修改、审计日志等功能
- **自动代码审查**：支持自动代码审查（Auto Code Review），在文件变更后自动运行 Code Review Agent 进行代码审查
- **LSP 工具**：新增完整的 LSP（Language Server Protocol）工具，支持跳转定义、查找引用、悬停提示、文档符号、工作区符号、跳转实现、调用层级分析等操作
- **API 使用量仪表板**：新增 API 使用量可视化页面，展示 Token 使用量、成本、请求数的图表统计，支持按今日/本周/本月筛选
- **自定义 Plan 和 Code Review 模型**：支持在设置中为 Plan Agent 和 Code Review Agent 分别配置专用模型
- **Plan 内容直接展示**：在 call-agent-tool-result 中直接展示 Plan 文件内容，无需手动打开文件
- **Custom Tool 支持 package.json**：Custom Tool 现在支持使用 package.json 定义依赖，自动安装 npm/bun 依赖包，支持任意第三方库
- 支持自定义 Plan Agent 和 Code Review Agent 的 Model

### 修复

- 修复 AI Pricing Service 和 LSP Tool 的相关 Bug
- 修复 API Usage Tab 刷新问题
- 修复 OpenAI 兼容类型的路径处理（自动追加 /v1）

### 致谢

- 感谢 @qingzhongli 修复 OpenAI 兼容类型的路径处理
